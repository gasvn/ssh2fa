01 how it works

Type your 2FA once.
Then just ssh.

[01]

Add a host

Its ssh-config alias, your password, and your 2FA secret — stored in the macOS Keychain. A one-shot test login runs before saving.

[02]

It stays warm

The daemon keeps two SSH ControlMaster connections alive per host, and auto-recovers them after sleep, network changes, and reboots.

[03]

Connect instantly

Run ssh <alias> in your own terminal. It reuses a warm connection — no password, no code, no waiting.

02 features

Built for the connection you live in.

Stays logged in

Two warm ControlMaster connections per host mean every ssh, scp, and editor remote reuses an open channel — instant, with no second factor to type.

Answers 2FA for you

Submits your password and a fresh TOTP code at login — serialized so a shared Duo secret is never replayed inside its window.

Self-healing

Adopts live connections across restarts and rebuilds them after sleep / wake — zero re-login.

SLURM port forwarding

Optional: forward a local port to a running job's compute node (Jupyter, dashboards) with staleness detection.

Power-user honest

Universal binary, 450+ tests, secrets in the Keychain, no telemetry — and a clear-eyed security model.

03 is it for you?

Honest about the fit.

[ yes, if… ]

  • + A host makes you type a password + a 2FA code on every login (Duo passcode, Google-Authenticator-style TOTP).
  • + You want persistent, instant SSH to an HPC / SLURM cluster on a Mac.
  • + You're tired of fragile expect + oathtool dotfiles doing this by hand.

[ not for… ]

  • − Key-based logins — ssh-agent already handles those.
  • − Push-only 2FA (approve-on-phone, no typed code).
  • − A terminal replacement — you keep using your own terminal.

Security, stated plainly

SSH2FA stores your SSH password and TOTP secret in the macOS Keychain and submits the second factor for you. That's the convenience — but it means your "second factor" no longer protects you against someone with access to your unlocked Mac. Read the full security model before you rely on it.
