How it works

Type your 2FA once. Then just ssh.

A small background helper handles the tedious part, so your terminal stays exactly the way you like it.

1

Import your hosts

SSH2FA reads your ~/.ssh/config — enable hosts in one click, or add one and scan its 2FA QR. Credentials live in the macOS Keychain.

2

It stays warm

The daemon keeps a connection alive per host and auto-recovers it after sleep, network changes, and reboots.

3

Connect instantly

Run ssh <alias> in your own terminal — it reuses the warm connection. SSH2FA sets that up for you; you edit nothing.

Features

Far more than auto-login.

Everything you need to set up, stay connected, and run your cluster — from your Mac.

No-retype 2FA login

Keeps a warm ControlMaster per host and auto-submits your password + a fresh TOTP code — so every ssh, scp, and editor remote connects instantly.

Your terminal, zero config

Your own ssh reuses the warm master too — via one consented, backed-up line. It never touches your own Host blocks.

Self-healing

Adopts live connections on launch and rebuilds them after sleep, Wi-Fi changes, and reboots — zero re-login, automatically.

One-click import

Reads ~/.ssh/config to enable hosts in one click, and scans a 2FA QR to add the secret — no double entry.

SLURM tunnels + countdown

Forward a port to a job's compute node via a node picker, with a live squeue expiry countdown before your allocation ends.

Locked down

Secrets in the macOS Keychain, optional Touch ID lock, no telemetry — and an honest, documented security model.

Security, stated plainly

SSH2FA stores your SSH password and TOTP secret in the macOS Keychain and submits the second factor for you. That convenience means your "second factor" no longer protects you against someone with access to your unlocked Mac. Read the full security model before you rely on it.
