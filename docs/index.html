SSH2FA

Log into your 2FA-protected SSH hosts once, then stay connected — no more typing a Duo/TOTP code every ssh.

macOS menu-bar app · built for HPC / SLURM cluster logins · works with any host that asks for a password + a typed 2FA code

Download View on GitHub

What it does

Stay logged in

Keeps two SSH ControlMaster connections warm per host. Every ssh reuses them instantly — no code to type.

Answers 2FA for you

Submits your password + TOTP code at login, serialized so a shared Duo secret is never replayed.

Self-healing

Adopts live connections across restarts, and auto-recovers after sleep, network changes, and reboots — zero re-login.

SLURM port forwarding

Optional: forward a local port to a running job's compute node (Jupyter, etc.), with staleness detection when the node changes.

Install

Download SSH2FA.dmg from Releases, drag the app to /Applications, and launch it. On first run it installs its background helper and registers a LaunchAgent for you — then use Add Host to save a host's credentials (stored in your macOS Keychain).

If the build isn't notarized, macOS blocks it the first time — open it once via System Settings → Privacy & Security → “Open Anyway,” or run: 
xattr -dr com.apple.quarantine /Applications/SSH2FA.app

Prefer to build it yourself? A locally-built app has no Gatekeeper friction — see the build instructions.

Is it for me?

✅ Yes if…

A host makes you type a password + a 2FA code every login (Duo passcode, Google-Authenticator-style TOTP), and you want persistent, instant SSH on a Mac.

❌ Not for…

Key-based logins (ssh-agent already handles those), push-only 2FA (approve-on-phone, no typed code), or as a terminal — you still ssh in your own terminal.

Security, plainly

SSH2FA stores your SSH password and TOTP secret in the macOS Keychain and submits the second factor for you. That's the convenience — but it means your "second factor" no longer protects you against someone with access to your unlocked Mac. Read the full security model before you rely on it.
